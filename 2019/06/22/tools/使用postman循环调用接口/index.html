使用postman循环请求接口

Q: 什么时候可以用的到此功能?

S: 模拟添加数据,有时候在添加数据的时候,要添加你规定的数据,但是使用sql写数据会有一些无法避免的问题,比如:写sql添加数据不能添加日志等…
此时就需要循环调用接口来实现数据的添加

-

Q: 可以实现什么样的功能?

S: 可以读excel/json.来实现指定数据的添加,也可以生成随机数据,随机数据可以使用js实现

比如我们现在实现这个接口的数据添加功能

首先点击上面的Run按钮

下面的几个参数分别是:

我们这里使用的测试数据

在接口上传数据的时候,上传的字段名是根据json的键值来使用的,所以不需要在参数里面设置解析

在这里选择需要运行的接口,因为并不是此文件夹下所有接口都要运行的,所以要进行勾选,只运行所需的接口

在选择完成后点左下的蓝色按钮 Run(接口名)

红色是运行异常的接口数量(非200)

绿色是运行正常的接口数量

Run Summary 查看统计图

Export Results 导出测试结果(json数据)
